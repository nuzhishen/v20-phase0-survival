# OTT 运维 FAQ

## Q001 直播卡顿怎么排查？
ChunkID：ott_q001
FAQID：Q001
CDN厂商：阿里云 / 腾讯云 / 网宿
播放器版本：Android Player 5.8+
指标：首帧时间、缓冲率、码率切换次数
Q：用户反馈直播卡顿、画面一顿一顿，应该怎么排查？
A：按顺序检查 CDN 节点状态、用户网络带宽、播放器版本、源站码率和自适应码率策略。先看缓冲率是否超过 8%，再看 CDN 回源是否超时，最后确认播放器是否低于 5.8。

## Q002 EPG 节目单不更新
ChunkID：ott_q002
FAQID：Q002
CDN厂商：不涉及
播放器版本：全版本
指标：EPG 更新时间、缓存 TTL
Q：电视端节目单不更新，仍显示昨天的频道信息怎么办？
A：检查 EPG 聚合任务是否成功、Redis 缓存 TTL 是否超过 24 小时、终端是否拉取到最新 epg_version。必要时清理 EPG 缓存并重新触发聚合。

## Q003 转码失败导致黑屏
ChunkID：ott_q003
FAQID：Q003
CDN厂商：腾讯云
播放器版本：Android Player 5.6+
指标：转码队列长度、HLS 切片生成耗时
Q：用户打开频道后黑屏，但有声音，如何处理？
A：优先检查转码任务是否失败，确认 H.264 profile、音视频时间戳和 HLS 切片是否完整。若转码队列堆积超过 1000 个任务，需要暂停新任务并扩容转码 worker。

## Q004 播放器版本兼容问题
ChunkID：ott_q004
FAQID：Q004
CDN厂商：不涉及
播放器版本：Android Player 5.5 以下
指标：崩溃率、解码失败率
Q：部分老设备播放新频道崩溃，如何判断是不是播放器版本问题？
A：按 device_model 和 player_version 聚合崩溃率，若 Android Player 5.5 以下解码失败率明显升高，先提示升级播放器或切换兼容码流。

## Q005 CDN 回源超时
ChunkID：ott_q005
FAQID：Q005
CDN厂商：阿里云 / 网宿
播放器版本：全版本
指标：回源耗时、源站 5xx、命中率
Q：CDN 边缘节点命中率下降、回源超时，直播会有什么影响？
A：用户会出现首帧变慢、直播卡顿和缓冲率上升。应检查源站带宽、CDN 预热状态、URL 鉴权和边缘节点健康度。

## Q006 DRM 鉴权失败
ChunkID：ott_q006
FAQID：Q006
CDN厂商：不涉及
播放器版本：Android Player 5.7+
指标：license 请求成功率、401 比例
Q：付费频道提示无权限或播放失败，如何排查 DRM？
A：检查 license server 是否返回 401/403，确认用户订购状态、token 过期时间、设备指纹和 DRM 证书链。不要绕过鉴权直接下发播放地址。

## Q007 源站码率配置过高
ChunkID：ott_q007
FAQID：Q007
CDN厂商：腾讯云
播放器版本：全版本
指标：平均码率、弱网缓冲率
Q：弱网用户大量反馈卡顿，源站码率是否可能过高？
A：如果 1080P 码率超过 8Mbps 且弱网缓冲率超过 10%，应降低默认码率或优化自适应码率阶梯，先灰度到 5% 用户。

## Q008 频道切换慢
ChunkID：ott_q008
FAQID：Q008
CDN厂商：阿里云 / 腾讯云
播放器版本：Android Player 5.8+
指标：首帧时间、DNS 解析耗时
Q：用户切台慢、首帧时间超过 3 秒，优先看哪些指标？
A：优先看 DNS 解析耗时、CDN 首包时间、播放器缓冲策略和频道鉴权接口耗时。首帧超过 3 秒时需要分区域聚合。

## Q009 播放地址 403
ChunkID：ott_q009
FAQID：Q009
CDN厂商：网宿
播放器版本：全版本
指标：403 比例、URL 鉴权过期时间
Q：播放地址返回 403 是 CDN 故障吗？
A：不一定。先检查 URL 鉴权签名、过期时间、用户订购状态和 CDN 防盗链规则。只有多个区域同时 403 且签名正常时才升级 CDN 厂商排查。

## Q010 直播音画不同步
ChunkID：ott_q010
FAQID：Q010
CDN厂商：腾讯云
播放器版本：Android Player 5.9+
指标：音视频 PTS 差值、转码延迟
Q：直播出现音画不同步，声音比画面快，如何定位？
A：检查转码输出的音视频 PTS 差值、播放器同步策略、HLS 分片时间戳和源站推流时间戳。PTS 差值超过 300ms 时应回滚转码模板。
