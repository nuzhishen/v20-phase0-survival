# Phase 0 Decision Record

## D-001: Phase 0 按天组织

- 状态：已采用
- 决策：一个总仓库，每天一个独立目录，Day 6 负责集成。
- 原因：降低每日认知负担，同时避免周六临时搬代码和修路径。

## D-002: P1 才项目化重建

- 状态：已采用
- 决策：Phase 0 只沉淀原型、接口思想和基线测试。
- 原因：避免实验代码污染正式的 TMS Agent Runtime。

## D-003: Day 2 默认使用 Mock

- 状态：已采用
- 决策：5 次 Mock 测试为必做，最多 2 次真实小额度调用为可选。
- 原因：不让网络、额度和外部 Provider 波动阻塞训练。

## D-004: Day 3 手写 ReAct 核心循环

- 状态：已采用
- 决策：核心循环不依赖 LangGraph。
- 原因：必须能解释状态、动作、观察、终止和防循环策略。

## D-005: Day 4 限制 RAG 范围

- 状态：已采用
- 决策：只实现切片、Embedding、Upsert、Search 和 Top-K。
- 原因：Phase 0 验证基础链路，不建设复杂 RAG 平台。

## D-006: 后续流式协议统一为 SSE

- 状态：已采用
- 决策：删除 WebSocket 实做计划。
- 原因：当前业务以服务端单向状态和结果推送为主，SSE 足够且更简单。
