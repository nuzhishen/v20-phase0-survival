## B. 公式推导步骤清单(手推时的骨架,不给答案)

### B1. BM25 —— 从三个直觉拼出打分公式
按顺序自己推,推不出来再看:

1. **起点:TF(词频)**。一个词在文档里出现越多越相关 —— 但**不能线性增长**(出现 100 次不该是 10 次的 10 倍相关)。→ 需要一个"**饱和函数**"。
2. **引入 k1**:用 `TF·(k1+1) / (TF + k1)` 形式让词频**饱和**。想清楚:k1→0 会怎样?k1→∞ 会怎样?(退化边界)
3. **引入 IDF(逆文档频率)**:越稀有的词区分度越高 → 权重越大。写出 `log((N - df + 0.5)/(df + 0.5) + 1)`,想清楚为什么要 +0.5 平滑、为什么用 log。
4. **引入 b 和文档长度归一化**:长文档天然更容易堆词频,要惩罚 → `(1 - b + b·|D|/avgdl)`。想清楚:b=0 会怎样(不归一化)?b=1 会怎样(完全归一化)?
5. **拼成完整式子**:`Σ_term IDF(t) · [TF·(k1+1)] / [TF + k1·(1-b+b·|D|/avgdl)]`。
6. **降级链**:BM25 卡住 → TF-IDF → 关键词命中计分(兜底,不许停工)。

### B2. 三种融合 —— 按"是否依赖分数尺度"分两类
1. **加权求和(需归一化)**:先 `normalize()` 两路分数到同尺度,再 `α·dense + (1-α)·sparse`。想清楚:不归一化会怎样(尺度不一致,一路吞掉另一路)?
2. **线性插值**:形式同加权,但强调 α 是"两个概率/分数分布之间插值",要写清和加权求和的语义差异(何时把它当同一式、何时区分)。
3. **RRF(不依赖分数,只依赖排名)**:`Σ_m 1/(k + rank_m(d))`,k=60。想清楚:为什么用 rank 而不用 score?为什么这天然免疫"Dense 分数 0~1、BM25 分数 0~30"的尺度不一致?

### B3. Reranker 成本 —— 从结构差异推延迟差异
1. **Bi-Encoder(双塔)**:query 和 doc **各自独立**编码成向量 → doc 向量**可离线预计算、可建索引** → 检索时只算一次 query 向量 + ANN。这就是 Day4 的检索器。
2. **Cross-Encoder(交叉)**:`[query, doc]` **拼接成一条**送进模型,层层**交叉注意力** → 输出一个相关性分数。→ **无法预计算 doc**、**每个 (query,doc) 对都要跑一次完整前向**。
3. **推成本**:N 个候选 = N 次完整前向 → 只能对 TopN(默认 ≤20)做,不能全库。→ Reranker 是"精排"不是"召回"。

| 1 | **为什么 Dense 不够?纯向量不是已经很强了吗?** | 逼你承认向量的结构性短板 | 对**错误码 `E1001` / 设备型号 / `Android 11` / 精确术语**召回弱 —— 向量会把编号"语义模糊化"。 |

| 3 | **Hybrid 权重 α 怎么定?凭感觉 0.5?** | 抓"拍脑袋"没有实验 | **不拍脑袋**:用 30 条 Query 跑 α=0.2/0.4/0.5/0.6/0.8 grid search,数据说话,并记录退化 Query。 |
| 4 | **RRF 凭什么有用?不就是加个排名倒数?** | 逼你讲清尺度不一致问题 | RRF **基于排名融合、不依赖分数尺度** —— 免疫"Dense 0~1 / BM25 0~30"的量纲冲突,鲁棒。 |
| 5 | **Reranker 这么准,直接拿它当检索器不就行了?** | 逼你分清召回 vs 精排 | Cross-Encoder **每对都要完整前向、无法预计算**,成本 O(N);只适合对 TopN 精排,不能全库召回。 |
| 6 | **为什么今天不做 Query Rewrite?加上不是更好?** | 逼你守住实验纪律 | 会**引入新变量、污染 Dense vs Hybrid 对比实验** —— 一次只动一个变量,归因才干净。 |
| 7 | **为什么不上 GraphRAG?现在很火。** | 逼你守住 Phase 0 边界 | Phase 0 目标是**通关闭环**,不是扩展知识图谱;GraphRAG 是复杂度扩张,不解决今天要证明的问题。 |

**附加高频追问(准备到能脱稿)**:
- "Hybrid 一定比 Dense 好吗?" → **不一定,会退化**;某些纯语义 Query 掺入 Sparse 反而引噪声,必须记录退化率。
